Self-hosted agent control

A self-hosted iPhone control surface for local agent work.

MOBaiLE is not a hosted IDE. It pairs your iPhone with a backend on your own Mac or Linux machine, then lets the phone start, follow, and continue agent runs.

What is a self-hosted iPhone agent control surface?

In MOBaiLE, the iPhone controls the run and the paired backend does the work. The backend lives on a host you operate, so your execution context stays attached to your repo, shell, files, credentials, network access, agent CLI, and host policy.

Host-owned runtime

The backend owns execution, pairing, auth, access mode, runtime state, run history, and file/runtime boundaries.

Phone-owned interaction

The phone captures prompts, voice, attachments, progress, summaries, artifacts, and follow-up.

Proof without private logs

mobaile demo exports sanitized activity-event replays for public proof without raw logs, prompts, file paths, or tokens.

Who this is for

  • Developers who already use local Codex or Claude Code workflows.
  • Founders and operators who need to start or follow work away from the desk.
  • Security-conscious users who want inspectable host-side boundaries.

First success

  1. Install the backend on the host you trust.
  2. Pair the iPhone with the backend QR.
  3. Run mobaile first-run.
  4. Watch planning, execution, and result return to one live phone thread.
