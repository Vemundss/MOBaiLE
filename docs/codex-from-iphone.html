Codex from iPhone

Run Codex from iPhone while your Mac does the work.

MOBaiLE pairs your iPhone with a Mac or Linux host so you can start Codex work by text or voice, follow progress live, and keep the run in one phone thread.

Can you run Codex from iPhone?

Yes. MOBaiLE lets the iPhone start and follow Codex CLI runs on a paired Mac or Linux host. The phone does not execute code. Your host keeps the repo, shell, credentials, files, network access, and signed-in Codex CLI environment.

Use the real repo

Codex runs where your work already lives, instead of requiring a copied prompt, uploaded project, or hosted development workspace.

Follow the run live

Planning, execution, blocked states, results, artifacts, and follow-up stay in the same phone-visible thread.

Keep access explicit

MOBaiLE keeps safe mode and full-access mode visible so you know how much authority the host runtime has.

Setup path

  1. Install MOBaiLE on the Mac or Linux host you want Codex to use.
  2. Make sure Codex CLI is installed and signed in on that host.
  3. Pair the iPhone with the QR shown by the backend.
  4. Run mobaile first-run to prove the live thread.
  5. Use mobaile demo --out mobaile-demo.md for a sanitized proof artifact.

Common questions

  • Does the iPhone run Codex? No. The paired host runs Codex.
  • Do I need credentials on the phone? The host keeps the signed-in CLI environment.
  • Can I share a demo? Yes. Demo exports are built from backend activity events and omit raw logs, prompts, paths, and tokens by default.
