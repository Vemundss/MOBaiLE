Can you run Claude Code from iPhone?

Yes. MOBaiLE lets the iPhone start and follow Claude Code runs on a paired host. The host keeps the Claude Code CLI, repo, shell, credentials, files, and network access. The iPhone captures prompts, voice, attachments, progress, and follow-up.

Phone-native control

Start a task by voice or text, follow progress, and send the next prompt without reopening the laptop.

Host-side execution

Claude Code runs on the paired computer, where the project, shell, credentials, and local tools already exist.

Inspectable trust model

Pairing, bearer auth, access mode, and network path are documented in the public backend repo.

Setup path

  1. Install MOBaiLE on the host you want Claude Code to use.
  2. Make sure Claude Code is installed and signed in on that host.
  3. Pair the iPhone with the backend QR.
  4. Run mobaile first-run to confirm live progress.
  5. Use mobaile demo when you need a sanitized replay.

Common questions

  • Does MOBaiLE replace Claude Code? No. It controls the host-side Claude Code path.
  • Does the phone need repo access? No. The host keeps the repo and file access.
  • Can the backend run in safe mode? Yes. Use safe mode for cautious hosts and full-access mode only for trusted private machines.
